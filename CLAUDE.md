## Project

Shared movie & TV recommender for two-person households backed by Trakt integration and AI recommendations.

## Tech Stack

- **Frontend**: Flutter 3.11+ (Dart) with Material 3 dark theme
- **State management**: Riverpod 2.5 (providers + StateNotifier)
- **Routing**: go_router 14.2 with typed routes
- **Backend**: Firebase (Auth, Firestore, Cloud Functions, Cloud Messaging)
- **Cloud Functions**: Node 22 TypeScript, firebase-admin 13, firebase-functions 7, Anthropic SDK 0.30
- **External APIs**: 
  - Trakt (watch tracking, OAuth 2.0, history/ratings/recommendations)
  - TMDB (movie/TV metadata)
  - Claude API (Phase 7+, taste profiles)
  - Reddit JSON (Phase 7+, social proof)
- **Storage**: SharedPreferences (device-level mode toggle), Firestore (all household data)

## Key Conventions

### State Management
- **Providers**: `Provider<T>` for singletons, `StreamProvider<T>` for Firestore listeners, `FutureProvider<T>` for one-shot async
- **StateNotifier**: Used only for mutable user state (`ModeController`)
- **Naming**: `*Provider` for providers, `*Service` for singletons, `*Controller` for StateNotifier
- **Autodipose**: StreamProviders handling screen-scoped data use `.autoDispose` to clean up listeners on pop

### Screen/Widget Structure
- **Screens**: Consume providers via `ref.watch()`, emit side effects via `ref.read().method()`
- **Shared widgets**: Stateless except `ScaffoldWithNavBar` (manages share intent listeners at root)
- **Error handling**: `AsyncErrorView` + per-provider error states + global error widget fallback
- **Bottom sheet**: `showModalBottomSheet()` for confirm dialogs; persistent on navigation
- **Share-to-Save** (Phase 4): `warm_start` + `cold_start` listeners in `ScaffoldWithNavBar` parse Android SEND intents

### External API Integration
- **Trakt**: Client ID as `--dart-define TRAKT_CLIENT_ID`, secret only in Firebase Secrets Manager
  - OAuth browser flow via `flutter_web_auth_2`, CSRF state saved to Firestore
  - Token refresh auto-triggered on 401; revoke on household leave
  - Callable proxies: `traktExchangeCode`, `traktRefreshToken`, `traktRevoke` in Cloud Functions
- **TMDB**: Key via `--dart-define TMDB_API_KEY`, fetches metadata on demand
- **Claude**: Async callable `concierge` + `generateTasteProfile`, AI blurbs cached in `/recommendations`

### Secret Handling
- Keys passed via `--dart-define-from-file=env.json` (build-time) or Firebase Secrets Manager (runtime)
- `env.json` gitignored; use `env.example.json` template
- Trakt `client_secret` lives **only** in `TRAKT_CLIENT_SECRET` Firebase secret
- No hardcoded secrets; all validated at startup

## Known Gotchas

1. **Cloud Functions region**: All CF callables explicitly set to `europe-west2` (London) to co-locate with Firestore; client code uses `FirebaseFunctions.instanceFor(region: 'europe-west2')` â€” do not default to us-central1
2. **Firestore model parsing**: Hardened to skip missing fields and map null â†’ sensible defaults; see `fromDoc` factory methods. Partial documents from share-parser are normal
3. **Trakt token refresh**: Happens auto on 401; client must catch and retry. Token stored encrypted at member doc â€” check `firebase_auth_mocks` in tests for token mocking
4. **Two-person cap**: Enforced in service layer (line-of-code comment in rules acknowledges Firestore can't count siblings easily)
5. **Share-to-Save listeners**: Cold-start listener set up in `main.dart`, warm-start in `ScaffoldWithNavBar`; both parse Android SEND intents asynchronously
6. **Phase 7+ AI blurbs**: `Recommendation` doc has dual fields (`ai_blurb` for together, `ai_blurb_solo` for solo mode); only populated once taste engine runs
7. **TMDB genres**: Hardcoded map in `utils/tmdb_genres.dart` â€” update if TMDB API schema changes
8. **Debug keystore**: Pinned in `android/app/build.gradle`; SHA-1 verified in CI â€” restores on each checkout to keep Google Sign-In consistent
9. **ExpansionTile + flag-gated nested Consumer**: avoid `children: [if (_flag) Consumer(...)]`. The Consumer never observes the FutureProvider's resolution because the rebuild after `setState` races the tile's expansion animation; symptom is a spinner that never resolves. Instead split into a sticky-flag wrapper + separate `ConsumerWidget` body â€” see `_GenreRow`/`_GenreRowBody` in `discover_screen.dart`.
